# domain/models/item

Item 集約っていうのは本当に商品だけに関心を当ててる。
だから、本当は、「注文」からItemNameを取り出して、、、とかの流れなんだけど、
とにかく、Item が存在することだけに注目する。だから、ItemNameはコンストラクタの引数にすればいい。

DBでいえば、Item Table を作るような感じ。ただし、カラムはコンストラクタの引数ではなく、
@propertyでアクセスできるやつが、カラムです。コンストラクタの引数は、カラムーチョを生成するための材料。
コンストラクタの引数は、その概念が誕生するときに必要な情報。
ある概念が誕生したときにそこに関わっていた情報がコンストラクタの引数になるべき。
